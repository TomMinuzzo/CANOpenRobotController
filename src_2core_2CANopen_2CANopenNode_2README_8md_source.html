1 CANopenNode
2 ===========
3 
4 CANopenNode is free and open source CANopen Stack.
5 
6 CANopen is the internationally standardized (EN 50325-4)
7 ([CiA301](http://can-cia.org/standardization/technical-documents))
8 CAN-based higher-layer protocol for embedded control system. For more
9 information on CANopen see http://www.can-cia.org/
10 
11 CANopenNode is written in ANSI C in object-oriented way. It runs on
12 different microcontrollers, as standalone application or with RTOS.
13 Stack includes master functionalities. For Linux implementation with
14 CANopen master functionalities see
15 https://github.com/CANopenNode/CANopenSocket.
16 
17 Variables (communication, device, custom) are ordered in CANopen Object
18 Dictionary and are accessible from both: C code and from CAN network.
19 
20 CANopenNode homepage is https://github.com/CANopenNode/CANopenNode
21 
22 
23 CANopen Features
24 ----------------
25  - NMT slave to start, stop, reset device. Simple NMT master.
26  - Heartbeat producer/consumer error control.
27  - PDO linking and dynamic mapping for fast exchange of process variables.
28  - SDO expedited, segmented and block transfer for service access to all parameters.
29  - SDO master.
30  - Emergency message.
31  - Sync producer/consumer.
32  - Non-volatile storage.

35 Usage of the CANopenNode
36 ------------------------
37 CANopenNode itself doesn't have complete working code for any microcontroller.
38 It is only the library with the stack and drivers for different
39 microcontrollers. It has example, which should compile on any system with
40 template driver (drvTemplate), which actually doesn't access CAN hardware.
41 CANopenNode should be used as a git submodule included in a project with
42 specific hardware and specific application.

113 File structure
114 --------------
115  - **CANopen.h/.c** - Initialization and processing of CANopen objects. Most
116  usual implementation of CANopen device.
117  - **stack** - Directory with all CANopen objects in separate files.
118  - **CO_Emergency.h/.c** - CANopen Emergency object.
119  - **CO_NMT_Heartbeat.h/.c** - CANopen Network slave and Heartbeat producer object.
120  - **CO_HBconsumer.h/.c** - CANopen Heartbeat consumer object.
121  - **CO_SYNC.h/.c** - CANopen SYNC producer and consumer object.
122  - **CO_SDO.h/.c** - CANopen SDO server object. It serves data from Object dictionary.
123  - **CO_PDO.h/.c** - CANopen PDO object. It configures, receives and transmits CANopen process data.
124  - **CO_SDOmaster.h/.c** - CANopen SDO client object (master functionality).
125  - **CO_trace.h/.c** - Trace object with timestamp for monitoring variables from Object Dictionary (optional).
126  - **crc16-ccitt.h/.c** - CRC calculation object.
127  - **drvTemplate** - Directory with microcontroller specific files. In this
128  case it is template for new implementations. It is also documented, other
129  directories are not.
130  - **CO_driver.h/.c** - Microcontroller specific objects for CAN module.
131  - **eeprom.h/.c** - Functions for storage of Object dictionary, optional.
132  - **helpers.h/.c** - Some optional files with specific helper functions.
133  - **socketCAN** - Directory for Linux socketCAN interface.
134  - **PIC32** - Directory for PIC32 devices from Microchip.
135  - **PIC24_dsPIC33** - Directory for PIC24 and dsPIC33 devices from Microchip.
136  - **dsPIC30F** - Directory for dsPIC30F devices from Microchip.
137  - **eCos** - Directory for all devices supported by eCos RTOS.
138  - **SAM3X** - Directory for SAM3X ARM Cortex M3 devices with ASF library from Atmel.
139  - **STM32** - Directory for STM32 ARM devices from ST.
140  - **LPC177x_8x** - Directory for LPC177x (Cortex M3) devices with FreeRTOS from NXP.
141  - **MCF5282** - Directory for MCF5282 (ColdFire V2) device from Freescale.
142  - **codingStyle** - Description of the coding style.
143  - **Doxyfile** - Configuration file for the documentation generator *doxygen*.
144  - **Makefile** - Basic makefile.
145  - **LICENSE** - License.
146  - **README.md** - This file.
147  - **example** - Directory with basic example.
148  - **main.c** - Mainline and other threads - example template.
149  - **application.h/.c** - Separate file with some functions, which are
150  called from main.c. May be used for application specific code.
151  - **CO_OD.h/.c** - CANopen Object dictionary. Automatically generated file.
152  - **IO.eds** - Standard CANopen EDS file, which may be used from CANopen
153  configuration tool. Automatically generated file.
154  - _ **project.xml** - XML file contains all data for CANopen Object dictionary.
155  It is used by *Object dictionary editor* application, which generates other
156  files. *Object dictionary editor* is currently fully functional, but old
157  web application. See http://sourceforge.net/p/canopennode/code_complete/.
158  - _ **project.html** - *Object dictionary editor* launcher.

186 ### Note for contributors
187  - Implementations for some other microcontrollers may go into CANopenNode.
188  - Each implementation should have a basic example, which should run on
189  implemented microcontroller.
190  - Use drvTemplate for template. Another template may be PIC32 microcontroller,
191  code for it is most maintained.
192  - Remove documentation comments from CO_driver.h (from drvTemplate). Make
193  only basic comments, same as in PIC32.
194  - Keep styling in CO_driver.(ch) unchanged, so files are easy comparable.
195  - In case of some (minor) changes in CANopenNode interface, change will
196  be updated for all microcontrollers.
